Cierre del caso práctico

Conclusiones y preguntas finales

El cierre resume por qué virtualizar, qué aporta cada modalidad y cómo llevar esta arquitectura a un proyecto real.

Optimización

Mejor uso de CPU, memoria y almacenamiento para cargas de IA y datos.

Portabilidad

Snapshots, backups, plantillas y contenedores facilitan mover servicios entre hosts.

Escalabilidad

Permite crecer por nuevas VMs, nuevos contenedores o más nodos de infraestructura.

1. ¿Cuál fue el motivo principal para usar virtualización en ciencia de datos e IA?

Optimizar recursos físicos, reducir costos, mejorar escalabilidad y ejecutar varias cargas como bases de datos, entrenamiento de modelos y visualización sin depender de hardware dedicado.

2. ¿Por qué la paravirtualización requiere modificar el sistema operativo invitado?

Porque el sistema invitado debe cooperar con el hipervisor mediante hypercalls o controladores específicos. Esto reduce overhead, baja latencia y mejora el rendimiento.

3. ¿Qué limitaciones tienen los contenedores?

Comparten el kernel del host, por lo que ofrecen menor aislamiento que una VM completa y dependen de la compatibilidad del sistema operativo base.

4. ¿Cómo garantizar la portabilidad futura?

Usando backups, snapshots, plantillas de VMs, imágenes Docker, repositorios de configuración y orquestadores como Kubernetes para mover servicios a otros hosts o nubes.

5. ¿Qué aprendizajes aplican a un proyecto real?

Administración de entornos virtualizados, asignación correcta de recursos, balanceo de carga, seguridad, monitoreo, alta disponibilidad y documentación de arquitectura.
