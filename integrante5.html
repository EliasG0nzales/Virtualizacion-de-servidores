Rol: expositor de contenedores y virtualización a nivel de sistema operativo.

Tema asignado: Docker, LXC, contenedores, microservicios y APIs de IA.

Descripción: explica cómo los contenedores comparten el kernel del host para lograr despliegues más rápidos y ligeros.

Debe destacar: diferencias VM vs contenedor, límites de recursos, portabilidad y arquitectura de microservicios.

Qué debe exponer

Los contenedores comparten el kernel del sistema operativo host, pero aíslan procesos, dependencias y configuraciones. Por eso son más ligeros y rápidos que una VM completa.

Evidencias

Guion para exponer

“Los contenedores son ideales para microservicios y APIs de IA porque arrancan rápido y consumen menos recursos. Comparten el kernel del host, lo que los hace más ligeros, aunque su limitación es que ofrecen menor aislamiento que una máquina virtual completa.”

Características de los contenedores

Arranque rápido: ideal para levantar servicios en segundos.

Bajo consumo: no carga un SO completo por servicio.

Portabilidad: una imagen puede ejecutarse en distintos hosts compatibles.

Escalado: se pueden replicar contenedores según demanda.

Menor aislamiento: comparten kernel con el host.

Microservicios: separa API, web, monitoreo y base de datos.
