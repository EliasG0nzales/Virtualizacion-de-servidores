Qué es virtualización completa

Es una modalidad donde cada máquina virtual funciona como un servidor independiente con su propio sistema operativo invitado. El hipervisor simula el hardware y reparte CPU, memoria, red y almacenamiento.

Recursos que se asignan

CPU: núcleos virtuales para procesamiento.

RAM: memoria para servicios y bases de datos.

Storage: discos virtuales, snapshots y datasets.

Red: interfaces virtuales y conectividad segura.

Evidencias

Guion para exponer

“La virtualización completa se usa cuando necesitamos aislamiento fuerte y compatibilidad. El hipervisor se instala sobre el hardware físico y permite crear varias máquinas virtuales, cada una con su propio sistema operativo. Para cargas críticas, se dimensionan CPU, RAM, red y almacenamiento, y luego se documentan políticas de alta disponibilidad.”

Dimensionamiento para servidores críticos

CPU: se asignan vCPU según la carga de procesamiento, consultas o entrenamiento.

Memoria: se reserva RAM suficiente para evitar intercambio excesivo a disco.

Almacenamiento: se separan discos para sistema, datos, logs y snapshots.

Red: se definen interfaces para administración, producción y migración.

Características de virtualización completa

Compatibilidad alta: permite instalar sistemas operativos completos sin modificar.

Aislamiento fuerte: cada VM tiene su propio kernel y configuración.

Snapshots: facilitan volver a un estado anterior de la VM.

Plantillas: permiten clonar servidores con configuración base.

Migración: puede integrarse con live migration o vMotion.

Overhead: consume más recursos que contenedores, pero ofrece más separación.
