Qué es paravirtualización

Es una modalidad donde el sistema operativo invitado sabe que está virtualizado y coopera directamente con el hipervisor. Esa cooperación reduce parte del costo de simular hardware completo.

Evidencias

Guion para exponer

“La paravirtualización mejora el rendimiento porque el sistema operativo invitado conoce que está virtualizado y coopera con el hipervisor. Mediante hypercalls o controladores específicos, se reduce overhead, latencia y uso innecesario de CPU.”

Flujo Host-Guest con hypercalls

Hypercalls: llamadas directas desde el invitado hacia el hipervisor.

Drivers específicos: optimizan red, disco y memoria.

Resultado: menor latencia y mejor eficiencia para cargas intensivas.

Características principales

Conocimiento del entorno: el invitado sabe que está virtualizado.

Menor emulación: evita simular todo el hardware.

Mayor rendimiento: útil para procesamiento intensivo.

Menor compatibilidad: no todos los sistemas invitados aplican.

Medición necesaria: se comparan CPU, RAM y latencia.

Uso recomendado: servicios controlados donde se pueda adaptar el SO.
